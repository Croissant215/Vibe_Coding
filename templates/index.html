🔍 취약한 vs 안전한 쿼리 비교

항목 취약한 방식 안전한 방식
쿼리 작성법 문자열 포맷팅 (f-string, format) Prepared Statement (?)
예제 f"SELECT * FROM users WHERE username = '{username}'" cursor.execute("SELECT * FROM users WHERE username = ?", (username,))
SQL Injection 공격 가능 ❌ 공격 차단 ✓
입력값 검증 필수 (하지만 불완전) 자동 처리 (신뢰할 수 있음)
권장도 사용 금지 권장

🎯 주요 차이점

1. 취약한 방식: 사용자 입력이 그대로 SQL 쿼리에 포함됨 → SQL 구문으로 해석됨 → 공격 가능

2. 안전한 방식: 입력값이 쿼리 매개변수로 전달됨 → 데이터로만 처리됨 → 공격 불가능

3. 예시: 입력값 ' OR '1'='1을 사용했을 때,

  • 취약함: ...WHERE username = '' OR '1'='1' → 모든 행 반환
  • 안전함: ...WHERE username = ? (? = 실제 문자열) → 데이터로 처리
