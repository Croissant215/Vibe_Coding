정상적인 사용자 인증 쿼리의 동작 방식

서비스에서 사용자가 입력한 아이디와 암호화된 비밀번호가 데이터베이스 조회(SELECT) 쿼리와 어떻게 결합되어 로그인 성공 여부를 판단하는지 확인합니다.

💡 핵심 개념: 일반적인 로그인은 WHERE username = '입력값' AND password_hash = '해시값' 조건에 일치하는 회원 레코드가 1개 이상 존재할 때 성립됩니다.
⚡ 빠른 테스트 입력 세팅
❓ 이해도 점검 퀴즈
🔍 실시간 쿼리 및 실행 결과 분석
대기 중
Executed SQL Query Vulnerable Mode
-- 왼쪽 입력 폼에서 아이디와 비밀번호를 입력 후 실행 버튼을 누르세요. -- 백엔드 데이터베이스에서 실행되는 조작된 SQL 문이 이곳에 표시됩니다.
💻 백엔드 구현 소스코드 비교 (secure.py)
def vulnerable_login(username, password): # ❌ 문자열 결합 방식 (f-string) 사용 - SQL Injection 취약! query = f"SELECT * FROM users WHERE username = '{username}' AND password_hash = '{password_hash}'" cursor.execute(query)
def secure_login(username, password): # ✅ Prepared Statement 사용 - 입력값을 단순 '데이터'로 처리 query = "SELECT * FROM users WHERE username = ? AND password_hash = ?" cursor.execute(query, (username, password_hash))
