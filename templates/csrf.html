1. 정상 계좌 송금 요청 실습

사용자 인증 세션 기반 송금 구조

로그인된 사용자가 정당하게 송금 폼을 작성하여 제출하는 정상적인 HTTP POST 백엔드 요청을 확인합니다.

💡 브라우저는 동일한 도메인으로 요청을 보낼 때 쿠키(Session Cookie)를 자동으로 포함합니다.

@app.route('/api/vulnerable-transfer') def vulnerable_transfer(): # ❌ CSRF 토큰 없이 쿠키 세션만으로 요청을 바로 승인! execute_transfer(recipient, amount)
@app.route('/api/secure-transfer') def secure_transfer(): # ✅ HTTP 헤더의 Anti-CSRF 토큰(X-CSRF-Token)을 정밀 검증 if request.headers.get('X-CSRF-Token') != session['csrf_token']: return jsonify({'error': '403 Forbidden'}), 403

💡 CSRF 공격 시뮬레이션 개요
CSRF(Cross-Site Request Forgery) 공격은 사용자가 로그인한 상태에서 악성 사이트를 방문했을 때 발생합니다. 악성 사이트 내부의 자동 제출 폼이 사용자의 세션 쿠키를 사용하여 불법 송금을 유발합니다.
