### 2️⃣ 취약한 로그인 (Vulnerable Login) ⚠️
**특징:** 해킹 공격에 노출됨!

#### 직접 해보기: 순서대로 따라하세요
1. 브라우저에서 "취약한 로그인" 탭 또는 섹션을 찾습니다.
2. 아이디 입력 칸에 `admin`을 넣고 비밀번호에 `admin123`을 넣습니다.
3. "로그인" 버튼을 누르면 정상 로그인 메시지가 나오는지 확인하세요.
4. 이제 아이디 칸을 꼭 복사해서 아래 공격 입력으로 바꿉니다.

#### 공격 시도 1: 모든 계정으로 로그인
```
아이디: ' OR '1'='1
비밀번호: 아무거나
```
- 위 문자열에는 작은따옴표(single quote) 4개가 있습니다.
- 작은따옴표가 빠지지 않도록, 아래 내용을 **정확히 그대로 복사**해서 아이디 칸에 넣으세요.
- 아이디 입력 칸에 다음을 그대로 붙여넣습니다:
  `' OR '1'='1`
- 다시 로그인 버튼을 누릅니다.
- 만약 성공한다면, 취약한 로그인 화면이 보안에 문제가 있다는 뜻입니다.
- 결과 메시지에서 "로그인 성공" 또는 계정 이름을 확인합니다.

#### 공격 시도 2: 비밀번호 우회
```
아이디: admin' --
비밀번호: 아무거나
```
- 다시 로그인 버튼을 누릅니다.
- 만약 성공하면, 비밀번호를 검사하지 않는 취약점이 있는 것입니다.

#### 왜 이렇게 해보는가?
- 이 페이지는 실제로 공격을 입력해서 "취약한지 아닌지" 확인하는 목적입니다.
- 공격 입력은 단순히 아이디 칸에 특수 문자를 넣어서 로그인 조건을 바꾸는 방법입니다.

**공격 원리:**
- 취약한 로그인 코드는 사용자가 입력한 것을 그대로 SQL 쿼리에 붙입니다.
- 그래서 컴퓨터는 입력을 코드로 읽습니다.
- 예를 들어:
  ```sql
  SELECT * FROM users WHERE username = '' OR '1'='1' AND password_hash = '...'
  ```
- `OR '1'='1'`은 항상 참이기 때문에, 비밀번호 검사 없이도 통과할 수 있습니다.

#### 더 쉬운 설명
- 1번 공격은 "항상 참인 조건"을 넣어서 통과하는 것.
- 2번 공격은 "비밀번호 검사 문장"을 주석 처리해서 없애는 것.

### 3️⃣ 안전한 로그인 (Secure Login) ✅
**특징:** 해킹 공격으로부터 완벽하게 보호됨!

#### 정상 사용:
```
아이디: admin
비밀번호: admin123
결과: ✅ 로그인 성공
```

#### 공격 시도 1: 모든 계정으로 로그인 시도
```
아이디: ' OR '1'='1
비밀번호: (아무거나)
결과: ❌ 로그인 실패 (아이디가 정확히 맞지 않음)
```

#### 공격 시도 2: 비밀번호 우회 시도
```
아이디: admin' --
비밀번호: (아무거나)
결과: ❌ 로그인 실패 (아이디가 정확히 맞지 않음)
```

**왜 안전할까?**
- 특수 문자(`'`, `--` 등)가 코드로 실행되지 않음
- 그냥 평범한 텍스트로 인식됨
- 아이디와 비밀번호가 정확히 일치해야 로그인됨

## 코드 이해하기

### 파일 구조
```
secure.py (이 파일)
├─ 데이터베이스 함수들
│  ├─ init_db() : 데이터베이스 만들기
│  └─ hash_password() : 비밀번호 암호화
├─ 회원가입 함수
│  └─ register_user() : 새 계정 등록
├─ 로그인 함수들
│  ├─ vulnerable_login() : 취약한 방식 ⚠️
│  └─ secure_login() : 안전한 방식 ✅
└─ 웹 페이지 (라우트)
   ├─ / : 메인 페이지
   ├─ /api/register : 회원가입 API
   ├─ /api/vulnerable-login : 취약한 로그인 API
   ├─ /api/secure-login : 안전한 로그인 API
   └─ /api/test-accounts : 테스트 계정 조회
```

### 핵심 개념 1: 비밀번호 암호화 (Hashing)
```python
def hash_password(password):
    # 입력: "admin123"
    # 출력: "240be518......" (64자 문자열)
    return hashlib.sha256(password.encode()).hexdigest()
```

**중요:** 
- 같은 비밀번호 = 같은 해시값
- 하지만 역으로는 불가능 (해시값에서 원본 비밀번호 알 수 없음)
- 해킹되어도 원본 비밀번호가 안 새어남

---

### 핵심 개념 2: SQL Injection 공격

#### ❌ 취약한 코드 (피해야 할 방식)
```python
query = f"SELECT * FROM users WHERE username = '{username}'"
```

**문제점:**
- 사용자 입력이 SQL 코드가 됨
- `username = "' OR '1'='1"` 를 입력하면:
  ```sql
  SELECT * FROM users WHERE username = '' OR '1'='1'
  ```

---

#### ✅ 안전한 코드 (추천하는 방식)
```python
query = "SELECT * FROM users WHERE username = ?"
cursor.execute(query, (username,))
```

**장점:**
- `?` 로 데이터 영역을 명확히 표시
- 사용자 입력이 데이터로만 처리됨 (코드 아님)
- `username = "' OR '1'='1"` 를 입력해도 그냥 일반 텍스트로 처리됨

## 자주 묻는 질문

### Q1: Flask는 뭐예요?
**A:** 파이썬으로 웹 서버를 만드는 도구입니다.
- `app.run(debug=True)` 로 서버 실행
- `@app.route('/')` 로 웹 페이지 만들기

### Q2: SQL Injection이 정말 위험한가요?
**A:** 매우 위험합니다! 
- 데이터 유출
- 데이터 삭제 또는 수정
- 권한 없이 관리자 권한 획득
- 프로덕션 서버에서는 절대 피해야 할 버그

### Q3: 실제 서비스에서 어떻게 방어하나요?
**A:** 여러 방법이 있습니다:
1. **Prepared Statement** (가장 중요)
   ```python
   cursor.execute("SELECT * FROM users WHERE id = ?", (user_id,))
   ```
2. **입력값 검증** (최소 길이, 문자 종류 확인)
3. **권한 최소화** (데이터베이스 사용자의 권한 제한)
4. **암호화** (민감한 정보 암호화)
5. **로깅** (의심스러운 활동 기록)

### Q4: 혼자서 더 학습하려면?
**A:** 다음을 추천합니다:
- 다양한 SQL Injection 공격 시도해보기
- 코드의 각 줄 주석 읽고 이해하기
- `hash_password()` 함수에 다양한 입력 시도
- 데이터베이스 파일(security_demo.db) 탐색

### Q5: 프로그램이 안 돌아가면?
**A:** 다음을 확인하세요:
1. Python 설치 확인: `python --version`
2. Flask 설치 확인: `pip list | grep flask`
3. Flask 설치: `pip install flask`
4. 포트 5000이 사용 중인지 확인
5. 파일이 UTF-8로 저장되었는지 확인

### Q6: `http://localhost:5000` 이 뭔가요?
**A:** 
- `localhost` = 현재 컴퓨터 (자기 자신)
- `5000` = 포트 번호 (창문 같은 개념)
- 로컬 서버에 접속하는 주소입니다

### Q7: 데이터는 어디 저장되나요?
**A:** `security_demo.db` 파일에 저장됩니다
- 같은 폴더에 자동 생성됨
- SQLite 데이터베이스 형식
- 프로그램을 다시 실행해도 데이터가 유지됨
